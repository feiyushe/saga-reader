# 前端展示域技术实现文档

## **1. 概述**

前端展示域是 Saga Reader 应用的用户交互入口，负责将系统核心功能（内容聚合、AI增强、本地搜索、主题切换）以直观、流畅、高响应性的界面呈现给用户。该域基于 **SvelteKit + TypeScript + Tailwind CSS** 构建，采用**组件化、响应式、状态驱动**的架构，实现了三栏式阅读工作区，是用户感知系统智能性的直接窗口。

本域不包含业务逻辑，其核心职责是：
- **渲染**：将状态管理域提供的数据以视觉化方式呈现
- **交互**：捕获用户操作（点击、输入、切换）并触发后端服务
- **反馈**：通过 UI 状态（加载、错误、空状态）提供即时反馈
- **一致性**：通过国际化与主题系统保障多语言、多环境下的视觉统一

前端展示域与状态管理域深度耦合，所有数据变更均通过 Svelte Store 驱动，所有用户操作均通过桥接层调用后端 Rust 服务，形成“UI → Store → Bridge → Rust → DB → Store → UI”的完整闭环。

## **2. 核心组件与功能实现**

前端展示域由**主界面组件**、**通用UI组件**和**配置与国际化**三类模块组成，共同构建完整的阅读体验。

### **2.1 主界面组件（Main Interface Components）**

> **路径**：`app/src/routes/main/widgets/` 
> **文件**：`FeedsList.svelte`, `ArticlesList.svelte`, `ArticleReader.svelte`, `SearchBar.svelte`, `Footer.svelte`, `AISpritePanel.svelte`, `ReaderBlankIndicator.svelte`

#### **2.1.1 FeedsList.svelte —— 订阅源树形导航**

- **功能**：渲染用户配置的所有订阅源（RSS/Bing/Baidu），支持分组折叠、点击选中、右键菜单（添加/删除/刷新）。
- **实现细节**：
  - 使用 `import { feeds } from '$lib/stores/feeds.svelte'` 订阅全局订阅源状态
  - 通过 `$: filteredFeeds = $feeds.filter(...)` 实现动态过滤（如按名称搜索）
  - 使用 `createEventDispatcher()` 派发 `selectFeed` 事件，通知父组件更新当前选中源
  - 利用 Svelte 的 `{#each}` 和 `{#if}` 实现树形结构的懒加载与展开/收起动画
  - 样式使用 Tailwind CSS 的 `group-hover` 和 `transition` 实现交互反馈

```svelte
<!-- FeedsList.svelte 片段 -->
<script>
  import { feeds } from '$lib/stores/feeds.svelte';
  import { createEventDispatcher } from 'svelte';
  const dispatch = createEventDispatcher();

  $: selectedFeedId = $reader.selectedFeedId;

  function handleFeedClick(feedId) {
    dispatch('selectFeed', feedId);
  }
</script>

<ul class=\"space-y-1\">
  {#each $feeds as feed (feed.id)}
    <li 
      class=\"px-3 py-2 rounded-md cursor-pointer transition-colors hover:bg-gray-100 dark:hover:bg-gray-700\"
      class:ring-2=\"{selectedFeedId === feed.id}\"
      class:ring-blue-500=\"{selectedFeedId === feed.id}\"
      on:click={() => handleFeedClick(feed.id)}>
      {feed.title}
    </li>
  {/each}
</ul>
```

#### **2.1.2 ArticlesList.svelte —— 文章列表与分组渲染**

- **功能**：根据当前选中订阅源，展示文章列表，支持按时间分组（今日、昨日、上周）、分页加载、搜索过滤、标记已读/收藏。
- **实现细节**：
  - 依赖 `import { articles } from '$lib/stores/articles/list/index.svelte'` 获取文章列表
  - 使用 `$derived` 计算分组逻辑（如 `groupedArticles = $derived(groupBy($articles, 'createdAt'))`）
  - 实现**懒加载**：监听滚动事件，当接近底部时调用 `featuresApi.loadMoreArticles()`
  - 搜索联动：通过 `import { filterText } from '$lib/stores/articles/search/index.svelte'` 实时响应搜索框变化
  - 使用 `ArticleRenderWidget.svelte` 组件复用文章项渲染逻辑

```svelte
<!-- ArticlesList.svelte 关键逻辑 -->
<script>
  import { articles } from '$lib/stores/articles/list/index.svelte';
  import { filterText } from '$lib/stores/articles/search/index.svelte';
  import { featuresApi } from '$lib/hybrid-apis/feed/impl';

  $: filteredArticles = $articles.filter(article => 
    article.title.toLowerCase().includes($filterText.toLowerCase()) ||
    article.purged_content?.toLowerCase().includes($filterText.toLowerCase())
  );

  async function loadMore() {
    await featuresApi.loadMoreArticles();
  }
</script>

{#if $loading}
  <div class=\"text-center py-4\">加载中...</div>
{:else if filteredArticles.length === 0}
  <ReaderBlankIndicator />
{:else}
  <div class=\"space-y-2\">
    {#each filteredArticles as article}
      <ArticleItem {article} />
    {/each}
  </div>
{/if}
```

#### **2.1.3 ArticleReader.svelte —— 多版本文章内容渲染**

- **功能**：渲染当前选中文章的四种版本：原始、净化（Purged）、优化（Optimized）、融合（Melted），支持一键切换。
- **实现细节**：
  - 通过 `import { reader } from '$lib/stores/reader.svelte'` 获取当前文章与版本标识
  - 使用 `{#if}` 条件渲染不同内容：`{#if $reader.version === 'melted'}`
  - 内容渲染使用 `Markdown.svelte` 组件，安全处理 HTML 与 Markdown
  - 支持**内容复制**、**保存为 PDF**（通过 `SaveOperatePanel.svelte`）
  - 使用 `on:click` 监听“AI助手”按钮，触发 `AISpritePanel` 显示

```svelte
<!-- ArticleReader.svelte 内容渲染片段 -->
<script>
  import { reader } from '$lib/stores/reader.svelte';
  import Markdown from '$lib/widgets/Markdown.svelte';
  import SaveOperatePanel from '$lib/widgets/SaveOperatePanel.svelte';

  const versions = [
    { label: '原始', value: 'original' },
    { label: '净化', value: 'purged' },
    { label: '优化', value: 'optimized' },
    { label: '融合', value: 'melted' }
  ];
</script>

<div class=\"prose dark:prose-invert max-w-none p-6\">
  {#if $reader.article}
    <Markdown>{getArticleContent()}</Markdown>
    <SaveOperatePanel {article} />
  {:else}
    <div class=\"text-center text-gray-500\">请选择一篇文章</div>
  {/if}
</div>

{#if $reader.version === 'melted'}
  <div class=\"mt-4 p-3 bg-blue-50 dark:bg-blue-900/30 rounded\">← 融合版：多源信息整合</div>
{/if}

{#if $reader.article}
  <button on:click=\"showAISpritePanel()\" class=\"btn btn-primary\">AI助手</button>
{/if}

{#if $isAISpriteVisible}
  <AISpritePanel />
{/if}
```

#### **2.1.4 AISpritePanel.svelte —— AI对话交互面板**

- **功能**：提供与当前文章相关的 AI 对话界面，支持历史记录、Markdown 渲染、输入发送。
- **实现细节**：
  - 依赖 `import { sprite } from '$lib/stores/sprite.svelte'` 获取对话历史与当前文章上下文
  - 使用 `featuresApi.chat_with_article_assistant()` 发起异步请求
  - 输入框使用防抖（debounce）机制，避免高频调用
  - 响应式更新：`$derived` 监听 `sprite.messages`，自动滚动到底部
  - 使用 `markdown-it` + `highlight.js` 渲染回复内容，支持代码块高亮

```svelte
<!-- AISpritePanel.svelte 核心交互 -->
<script>
  import { sprite } from '$lib/stores/sprite.svelte';
  import { featuresApi } from '$lib/hybrid-apis/feed/impl';
  import Markdown from '$lib/widgets/Markdown.svelte';

  let inputText = '';
  let isSending = false;

  async function handleSend() {
    if (!inputText.trim() || isSending) return;

    isSending = true;
    const message = { role: 'user', content: inputText };
    sprite.addMessage(message);

    try {
      const response = await featuresApi.chat_with_article_assistant(
        $sprite.currentArticleId,
        inputText,
        $sprite.messages
      );
      sprite.addMessage({ role: 'assistant', content: response });
    } catch (err) {
      sprite.addMessage({ role: 'assistant', content: 'AI服务出错，请重试。' });
    } finally {
      inputText = '';
      isSending = false;
    }
  }

  $: $: scrollIntoView(); // 自动滚动到最新消息
</script>

<div class=\"h-full flex flex-col\">
  <div class=\"flex-1 overflow-y-auto p-4 space-y-4\">
    {#each $sprite.messages as msg (msg.id)}
      <div class=\"{msg.role === 'user' ? 'text-right' : 'text-left'}\">
        <Markdown>{msg.content}</Markdown>
      </div>
    {/each}
  </div>
  <div class=\"p-4 border-t dark:border-gray-700\">
    <input 
      type=\"text\" 
      bind:value={inputText} 
      on:keydown={e => e.key === 'Enter' && handleSend()} 
      placeholder=\"提问关于这篇文章...\" 
      disabled={isSending} 
      class=\"w-full p-2 rounded border\" />
  </div>
</div>
```

#### **2.1.5 SearchBar.svelte —— 全文搜索输入框**

- **功能**：提供全局关键词搜索，实时过滤文章列表。
- **实现细节**：
  - 双向绑定至 `import { filterText } from '$lib/stores/articles/search/index.svelte'`
  - 使用 `setTimeout` 实现 300ms 防抖，避免频繁触发数据库查询
  - 搜索时显示加载动画，搜索结果为空时显示 `ReaderBlankIndicator`
  - 支持快捷键（Ctrl+K）快速聚焦

```svelte
<!-- SearchBar.svelte -->
<script>
  import { filterText } from '$lib/stores/articles/search/index.svelte';
  import { onMount } from 'svelte';

  let inputEl;

  onMount(() => {
    const handleKeyDown = (e) => {
      if (e.ctrlKey && e.key === 'k') {
        e.preventDefault();
        inputEl.focus();
      }
    };
    window.addEventListener('keydown', handleKeyDown);
    return () => window.removeEventListener('keydown', handleKeyDown);
  });

  function handleInput(e) {
    filterText.set(e.target.value);
  }
</script>

<div class=\"relative\">
  <input 
    bind:this={inputEl}
    type=\"text\" 
    bind:value={$filterText}
    on:input={handleInput}
    placeholder=\"搜索文章标题或内容...\" 
    class=\"w-full pl-10 pr-4 py-2 rounded border\" />
  <svg class=\"absolute left-3 top-2.5 h-5 w-5 text-gray-400\" fill=\"none\" stroke=\"currentColor\" viewBox=\"0 0 24 24\">
    <path stroke-linecap=\"round\" stroke-linejoin=\"round\" stroke-width=\"2\" d=\"M21 21l-6-6m2-5a7 7 0 11-14 0 7 7 0 0114 0z\"/>
  </svg>
</div>
```

### **2.2 通用UI组件（Reusable UI Widgets）**

> **路径**：`app/src/lib/widgets/`

| 组件 | 功能 | 技术实现 |
|------|------|----------|
| `Markdown.svelte` | 安全渲染 Markdown/HTML | 使用 `markdown-it` + `sanitize-html` 过滤脚本，支持代码高亮、图片懒加载 |
| `ContextMenuProvider.svelte` | 右键上下文菜单 | 使用 `svelte-context` + `on:contextmenu`，动态生成菜单项（复制、打开链接、标记已读） |
| `EmbedWebView.svelte` | 嵌入外部网页 | 使用 `<iframe>` + `tauri://` 协议安全加载，防止 XSS |
| `SaveOperatePanel.svelte` | 保存/取消按钮 | 使用 `svelte-store` 管理保存状态，调用 `featuresApi.save_article_as_pdf()` |
| `MarkdownImg.svelte` | 图片加载失败处理 | 使用 `on:error` 监听，显示占位图，支持重试 |

**示例：Markdown.svelte 安全渲染**

```svelte
<!-- Markdown.svelte -->
<script>
  import markdownIt from 'markdown-it';
  import sanitizeHtml from 'sanitize-html';

  export let content = '';

  const md = markdownIt({
    html: true,
    linkify: true,
    highlight: (str, lang) => {
      if (lang && hljs.getLanguage(lang)) {
        return hljs.highlight(str, { language: lang }).value;
      }
      return ''; // 无语言时返回空
    }
  });

  $: htmlContent = sanitizeHtml(md.render(content), {
    allowedTags: ['p', 'h1', 'h2', 'h3', 'h4', 'h5', 'h6', 'code', 'pre', 'a', 'img', 'strong', 'em', 'ul', 'ol', 'li', 'br', 'blockquote'],
    allowedAttributes: {
      a: ['href', 'target'],
      img: ['src', 'alt', 'title']
    }
  });
</script>

<div class=\"prose dark:prose-invert\" innerHTML={htmlContent} />
```

### **2.3 配置与国际化（Configuration & Internationalization）**

> **路径**：`app/src/lib/i18n/`, `app/src/lib/themes/`

#### **2.3.1 多语言支持（i18n）**

- **实现方式**：基于 `svelte-i18n` + JSON 语言包
- **语言包结构**：`en.json`, `zh-CN.json`，包含所有 UI 文本（如“刷新”、“AI助手”、“已读”）
- **动态切换**：
  - 用户在设置页修改语言 → `featuresApi.set_user_config({ locale: 'zh-CN' })`
  - 后端写入 `user_config.toml`
  - 前端 `i18n/settings.ts` 监听配置变更，调用 `setLocale()`
  - 所有组件自动重渲染，文本更新

```ts
// i18n/settings.ts
import { setLocale } from 'svelte-i18n';
import { userConfig } from '$lib/stores/user_config.svelte';

$: if ($userConfig.locale) {
  setLocale($userConfig.locale);
}
```

#### **2.3.2 主题管理（Theme）**

- **实现方式**：CSS 变量 + 类名切换
- **主题文件**：`themes/index.ts` 定义 `light` 和 `dark` 主题的 CSS 类
- **切换逻辑**：
  - 读取 `app_config.toml` 中的 `theme` 字段（`system`, `light`, `dark`）
  - 使用 `document.documentElement.classList.toggle('dark', isDark)`
  - 支持系统偏好自动匹配（`prefers-color-scheme`）

```ts
// themes/index.ts
export function applyTheme(theme) {
  const html = document.documentElement;
  html.classList.remove('light', 'dark');

  if (theme === 'dark' || (theme === 'system' && window.matchMedia('(prefers-color-scheme: dark)').matches)) {
    html.classList.add('dark');
  } else {
    html.classList.add('light');
  }
}

// 在 +layout.ts 中初始化
applyTheme($appConfig.theme);
```

## **3. 与其它域的交互模式**

前端展示域**不直接调用后端服务**，所有通信均通过**状态管理域**与**桥接层**完成，确保架构清晰与可测试性。

| 交互场景 | 调用链 | 说明 |
|----------|--------|------|
| 用户点击“刷新” | `FeedsList → dispatch('refresh') → Store.update_feeds() → featuresApi.update_feed_contents()` | 通过 Store 触发后端任务，UI 响应加载状态 |
| 用户输入搜索词 | `SearchBar → filterText.set() → articles/search/store → $derived → articles/list/store → featuresApi.search_articles()` | 搜索逻辑由 Store 自动触发，避免前端重复调用 |
| 用户发送AI提问 | `AISpritePanel → sprite.addMessage() → featuresApi.chat_with_article_assistant()` | 异步请求由组件直接调用桥接层，回复通过 Store 更新渲染 |
| 用户切换主题 | `SettingsPage → featuresApi.set_app_config() → config.store → themes/index.ts → document.documentElement.classList` | 配置变更由后端持久化，前端监听并应用 |
| 应用启动加载文章 | `+page.svelte → createStore() → articles/list/store → featuresApi.get_feeds_packages() → ArticleRecorderService.query()` | 初始化时从数据库加载数据，Store 作为唯一数据源 |

> ✅ **关键原则**：**UI 只负责渲染和事件，状态负责数据流，桥接负责通信**。

## **4. 性能与用户体验优化**

| 优化点 | 实现方式 | 效果 |
|--------|----------|------|
| **懒加载** | ArticlesList 滚动到底部时异步加载更多文章 | 减少初始加载时间，提升首屏体验 |
| **防抖搜索** | SearchBar 输入延迟 300ms 触发搜索 | 避免高频数据库查询，降低 CPU 负载 |
| **响应式渲染** | Svelte 响应式声明（`$:`）自动追踪依赖 | 仅重渲染变更部分，避免全量更新 |
| **骨架屏** | 加载时显示占位符（Skeleton） | 提升感知性能，减少用户焦虑 |
| **错误边界** | 全局 `onerror` 捕获未处理异常，显示友好提示 | 避免白屏，提升稳定性 |
| **缓存优化** | 文章内容在 Store 中缓存，避免重复请求 | 减少网络与数据库访问 |
| **无障碍支持** | 所有按钮添加 `aria-label`，支持键盘导航 | 符合 WCAG 标准，提升可访问性 |

## **5. 潜在风险与改进建议**

### **5.1 风险**

| 风险 | 说明 | 建议 |
|------|------|------|
| **前端状态复杂度高** | `index.svelte.ts` 超 200 行，多个 Store 耦合 | 拆分为独立 Store：`FeedStore`, `ArticleStore`, `AISpriteStore`，使用 `svelte/store` 模块化 |
| **样式污染风险** | Tailwind CSS 全局类可能冲突 | 引入 `svelte-css` 模块化样式，或使用 `:global()` 严格隔离 |
| **AI回复渲染性能** | 长文本 Markdown 渲染卡顿 | 使用 `web worker` 异步渲染，或分块渲染（chunked rendering） |
| **无离线状态提示** | 网络中断时，搜索/刷新无提示 | 增加 `networkStatus` Store，UI 显示“离线模式” |
| **未实现动画过渡** | 组件切换无过渡效果 | 使用 `svelte/animate` 实现淡入淡出、滑动效果 |

### **5.2 改进建议**

1. **组件拆分**：将 `ArticleReader.svelte` 拆分为 `ArticleHeader.svelte`, `ArticleContent.svelte`, `ArticleActions.svelte`，提升复用性
2. **类型增强**：为所有 Store 定义 TypeScript 接口，如 `interface ArticleStore { articles: Article[], loading: boolean }`
3. **测试覆盖**：为 `AISpritePanel.svelte` 编写 Svelte Testing Library 单元测试，验证输入发送与回复渲染
4. **性能监控**：集成 `performance.mark()` 记录组件渲染耗时，识别瓶颈
5. **文档化**：为每个组件编写 Storybook 示例，便于团队协作与新人上手

## **6. 总结**

前端展示域是 Saga Reader 的“门面”，它以**极简的 Svelte 组件**实现了**复杂的智能阅读体验**。其成功在于：

- **架构清晰**：严格遵循“UI → Store → Bridge → Rust”分层，职责单一
- **响应式驱动**：Svelte 的响应式系统让 UI 与数据同步零延迟
- **体验极致**：防抖搜索、懒加载、AI精灵、主题切换等细节打磨到位
- **扩展性强**：组件化设计支持快速新增功能（如“阅读模式”、“语音朗读”）

> **前端展示域不是功能的实现者，而是价值的呈现者**。它将 Rust 引擎的智能处理能力，转化为用户可感知、可交互、可信赖的阅读体验，完美支撑了 Saga Reader “本地AI阅读”的核心价值主张。
